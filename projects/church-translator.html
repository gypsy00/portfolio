Web Application

Church Translator

A real-time AI-powered sermon translation app that lets a preacher speak in English while congregation members receive live Ukrainian captions on their phones — no app downloads required.

↗ Live Demo </> Source Code

The Problem

In multilingual church services, congregation members who don't speak the preacher's language are left behind. Professional interpreters are expensive, and existing translation tools aren't built for live, continuous speech in a church setting.

I needed something that could capture speech in real-time, translate it instantly, and deliver it to multiple phones simultaneously — all through a browser, with no setup required from the audience.

How It Works

The app has two roles: a Broadcaster (the preacher) and Listeners (the congregation). The broadcaster's browser captures English speech using the Web Speech API, sends it to the server for AI translation via Claude, and the translated text is pushed to all connected listeners through Server-Sent Events.

Preacher's Browser ↓ Web Speech API (captures English) Next.js API Route /api/translate ↓ Claude Haiku (translates to Ukrainian) Broadcast Store (in-memory pub/sub) ↓ Server-Sent Events Listener's Phone (live captions)

Key Features

Live Speech Recognition Uses the browser's Web Speech API to capture continuous English speech with interim results displayed in real-time.
AI-Powered Translation Each complete sentence is sent to Claude Haiku for fast, context-aware English-to-Ukrainian translation.
Real-Time Broadcasting Translations are pushed to all connected listeners simultaneously via SSE, with a 15-second heartbeat to keep connections alive.
QR Code Joining The broadcaster displays a dynamic QR code that listeners scan to join — includes the local network IP for LAN use during services.
Progressive Web App Screen wake lock keeps phones awake during the service. Responsive dark UI optimised for mobile viewing.
Auto-Scrolling Captions Latest translations stay visible at the top, older captions fade down. No manual scrolling needed.

Tech Stack

Next.js 16 Full-stack framework — App Router for file-based routing, API routes for translation and SSE endpoints.
TypeScript Type safety across the entire codebase — shared types between client and server.
AI SDK + Claude Haiku Fast, cost-effective translation model accessed through Vercel's AI SDK for clean provider-agnostic code.
Server-Sent Events One-way real-time streaming from server to listeners. Lighter than WebSockets for this broadcast-only pattern.
Web Speech API Browser-native speech recognition — no third-party services needed for capturing the preacher's voice.
Tailwind CSS Utility-first styling for a responsive, dark-themed mobile interface.

What I Learned

This project taught me how to combine multiple real-time technologies into a single coherent experience. I learned when to choose SSE over WebSockets (when communication only flows one direction), how to work with AI APIs for real-time use cases where latency matters, and how to design for a specific user context — people holding phones during a church service.

I also gained experience deploying to Vercel and managing environment variables for API keys in production. The in-memory broadcast store works for a single server instance, but I've noted in the code where Redis pub/sub would be needed for multi-region scaling.
