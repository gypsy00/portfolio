Desktop Application

Student Track

A JavaFX desktop application for managing student course registrations, enrollments, and academic results — with business rule enforcement for credit limits, schedule clashes, and grade reporting.

↗ Watch Demo </> Source Code

The Problem

Academic institutions need systems that don't just store data but actively enforce rules. A student shouldn't be able to enrol in more credits than allowed. Two courses that clash in schedule shouldn't both be registerable. Grades shouldn't be recorded for students who aren't enrolled.

This project handles the complete lifecycle of academic administration — from student and course creation through enrollment with validation, to grade recording and summary reporting.

How It Works

The application follows a layered MVC architecture. JavaFX FXML files define the UI layout, controllers handle user interaction, a central business logic engine enforces all rules, and a storage layer persists data to CSV files. A Searchable interface provides real-time filtering across all views.

JavaFX UI (FXML + Controllers) ├── StudentView → add, update, delete, search ├── CourseView → manage courses with schedules ├── EnrollmentView → enrol with validation ├── ResultsView → record grades └── ReportsView → summaries and statistics ↓ Business Logic Engine ├── Credit limit check (max 70 credits) ├── Schedule clash detection ├── Duplicate enrollment prevention └── Enrollment verification for grades ↓ Storage Layer (CSV files) ├── students.txt ├── courses.txt ├── enrollments.txt └── results.txt

Key Features

Student Management Add, update, delete, and search students with email validation and duplicate student ID prevention.
Course Management Full CRUD for courses including credit value, day, time, and room — all validated on input.
Smart Enrollment Enforces a maximum of 70 credits per student, detects overlapping schedules on the same day, and prevents duplicate enrollments.
Grade Recording Grades on a 0–100 scale with a 40% pass mark. The system verifies a student is enrolled before allowing grade entry.
Reporting Student summaries showing enrolled courses, pass/fail counts, and average grades. Course-level statistics for staff.
Real-Time Search All table views support live filtering via a shared Searchable interface — type and results update instantly.

Tech Stack

Java 17 Core language. Object-oriented design with clean separation between domain models, logic, and UI.
JavaFX 17 Desktop UI framework with FXML for declarative layouts and CSS-like styling for the interface.
Maven Build system handling dependency management, compilation, and test execution.
JUnit 5 Unit testing for business logic — credit limit checks, schedule clash detection, and enrollment rules.
FXML XML-based UI layout files that separate view structure from controller logic.
CSV Storage Lightweight text-based persistence. Each entity type has its own file with structured read/write operations.

What I Learned

This project was all about business rule enforcement. The most interesting challenge was the schedule clash detection — I had to compare time ranges across all of a student's enrolled courses on the same day, accounting for overlapping periods rather than just exact matches.

Working with JavaFX taught me about event-driven programming in a desktop context — binding UI elements to data, handling user actions through controllers, and keeping the view layer completely separate from business logic. The Searchable interface pattern let me add real-time filtering to every view without duplicating code.

Writing JUnit tests for the business rules also reinforced the value of testing — I caught edge cases in the credit calculation and schedule overlap logic that I wouldn't have found through manual testing alone.
