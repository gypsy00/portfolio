Browser Game

Snack Man!

A Pac-Man inspired browser game with grid-based maze navigation, AI-controlled enemies, a scoring system, lives, and a persistent leaderboard — built entirely with vanilla JavaScript.

↗ Play Game </> Source Code

The Problem

I wanted to understand how games work at a fundamental level — game loops, collision detection, enemy AI, and state management — without relying on a game engine. The goal was to build something playable and fun using only what the browser gives you: HTML, CSS, and JavaScript.

How It Works

The game runs on a 10x10 CSS Grid where each cell is either a wall, a path with a dot, or empty. The player moves through the grid collecting dots while avoiding three enemies that move randomly. The game loop runs on a timed interval, updating enemy positions and checking for collisions each tick.

Game Loop (setInterval) ↓ each tick ├── Input Handler │ ├── Arrow keys (keyboard) │ └── On-screen buttons (touch/mobile) ├── Player Update │ ├── Move in current direction │ ├── Check wall collision │ └── Collect dot (+10 points) ├── Enemy Update (x3) │ ├── Random direction selection │ └── Wall-aware movement ├── Collision Check │ ├── Player vs Enemy → lose life │ └── All dots collected → win └── Render ├── Update grid cells ├── Update score / lives display └── Animate player mouth direction

Key Features

Grid-Based Gameplay 10x10 maze built with CSS Grid. Walls, paths, and dots are all defined in a map array that's easy to modify.
Enemy AI Three AI-controlled enemies that navigate the maze using random direction selection with wall awareness — they can't walk through walls.
Scoring System +10 points per dot collected. Total score displayed in real-time. Win condition: collect all dots on the grid.
Lives System 3 lives with visual indicators. Lose a life when an enemy catches you. Game over at 0 lives.
Persistent Leaderboard High scores saved to localStorage and displayed in a sidebar. Scores persist across browser sessions.
Mobile Controls On-screen directional buttons for touch devices alongside keyboard arrow key support for desktop.
Directional Animation The player character's mouth animates to face the current movement direction using CSS clip-path.

Tech Stack

Vanilla JavaScript Zero dependencies. Game loop, state management, collision detection, and enemy AI all written from scratch.
CSS Grid The entire game board is a CSS Grid. Each cell's visual state is controlled by adding/removing CSS classes.
CSS Animations Player mouth animation uses clip-path transforms. Smooth transitions for movement and visual feedback.
localStorage Persistent leaderboard storage. Scores are serialised to JSON and survive browser restarts.
HTML5 Semantic structure with a sidebar for leaderboard, centred game board, and control panels.
Google Fonts Press Start 2P retro font for the authentic arcade game aesthetic.

What I Learned

Building a game without a framework taught me how game loops actually work — the cycle of reading input, updating state, checking collisions, and rendering, all running on a fixed interval. It's a pattern that translates well to understanding any real-time system.

The collision detection was simpler than I expected because the grid-based approach means you only need to check if two entities occupy the same cell, rather than calculating pixel-perfect overlaps. This taught me that choosing the right data structure (a grid vs. free-form coordinates) can dramatically simplify your logic.

I also learned about the importance of input responsiveness — the game feels much better when directional changes are queued and applied on the next tick rather than processed immediately, which prevents the player from moving through walls by pressing keys too fast.
