GitHub Profile Analyzer

A terminal CLI tool that analyses any GitHub user's profile — showing language breakdowns with visual bars, top repositories ranked by stars, and recent activity tracking.

↗ View on npm </> Source Code

Try It Yourself

Open your terminal and run:

# Run instantly with npx (no install needed) npx gh-profile-peek gypsy00 # Or install globally npm install -g gh-profile-peek gh-peek gypsy00 # Analyse any GitHub user gh-peek torvalds gh-peek gaearon # Options gh-peek username --top 10 # show top 10 repos gh-peek username --token ghp_... # use a GitHub token for higher rate limits

Requires Node.js 18+. No GitHub token needed for basic use — but adding one with --token increases the rate limit from 60 to 5,000 requests per hour.

The Problem

GitHub profiles show basic stats, but getting a quick, formatted overview of someone's coding activity — what languages they actually use, which repos matter most, and what they've been working on recently — requires clicking through multiple pages.

I wanted a single command that pulls all of this together in the terminal, formatted with colour-coded bars and clean typography. Something you could run before a code review, a hiring decision, or just out of curiosity.

How It Works

The CLI takes a GitHub username as input, makes multiple API calls to fetch profile data, repository details, language breakdowns, and recent events. It then aggregates the data — combining language byte counts across up to 30 repos — and renders everything as formatted terminal output with coloured percentage bars and structured sections.

$ npx gh-profile-peek username ↓ Commander (parse args: username, --top, --token) ↓ API Layer (src/api.js) ├── GET /users/{username} ├── GET /users/{username}/repos (up to 30) ├── GET /repos/{owner}/{repo}/languages (per repo) └── GET /users/{username}/events (last 90 days) ↓ Analyze Layer (src/analyze.js) ├── Aggregate language bytes across all repos ├── Rank repos by star count └── Group events by type (Push, PR, Issue, etc.) ↓ Display Layer (src/display.js) ├── Profile card (name, bio, location, followers) ├── Language bars (coloured percentages) ├── Top repos table (stars, forks, language) └── Activity summary (event counts by type)

Key Features

Language Breakdown Aggregates byte counts from up to 30 repos and displays them as coloured percentage bars in the terminal.
Top Repositories Ranked by star count with fork counts and primary language shown. Configurable number of repos via --top flag.
Activity Tracking Groups the last 90 days of public events by type — PushEvent, PullRequestEvent, IssueCommentEvent, etc.
Profile Overview Displays name, bio, location, company, follower/following counts, public repos, and account age.
Token Support Accepts a GitHub Personal Access Token via --token for higher API rate limits (5,000 vs 60 requests/hour).
npm Publishable Configured with a bin entry point — installable globally or runnable directly with npx.

Tech Stack

Node.js 18+ Runtime with native fetch API — no HTTP library needed for GitHub API calls.
Commander CLI argument parsing — handles positional args (username) and flags (--top, --token).
Chalk v5 Terminal colour library (ESM). Renders coloured bars, highlighted text, and structured output.
Ora Loading spinners for API calls — gives users visual feedback while data is being fetched.
GitHub REST API v3 Data source for profiles, repos, languages, and events. Handles pagination and rate limiting.
ESM (Pure) Full ES Module setup with "type": "module" in package.json. No CommonJS, no build step.

What I Learned

This was my first time building a distributable CLI tool. I learned how npm's bin field works, how shebang lines let Node scripts run as executables, and how to structure a CLI with clean separation between argument parsing, API calls, data processing, and terminal rendering.

Working with the GitHub API taught me about rate limiting and how to handle it gracefully — the tool works without authentication but warns users about the lower limit and accepts a token for power users.

The language aggregation was interesting because GitHub's language API returns byte counts per repo, not percentages. I had to sum bytes across all repos and calculate percentages myself, then map them to a fixed-width bar that renders cleanly in any terminal width.
